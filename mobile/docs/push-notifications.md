# Push Notifications — Implementation Guide

May 24 2026

This document captures the full push notification stack used in CourtFlow mobile and can serve as a blueprint for reuse in another Expo + Next.js (or any Node.js backend) project.

---

## Architecture overview

```
Server (firebase-admin)
  → FCM data-only message
    → Android: Expo's ExpoFirebaseMessagingService
      → expo-notifications JS layer (token, handlers, tap listeners)
    → iOS: APNs (via expo-notifications)
```

**There is no direct `@react-native-firebase/messaging` or Firebase JS SDK on the client.** `expo-notifications` acts as the abstraction layer over both FCM (Android) and APNs (iOS). The server uses `firebase-admin` to send to FCM.

**Expo Go is not supported** — a dev client or standalone build is required.

## 2. Firebase project setup

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com).
2. Add an **Android app** with your app's package name (e.g. `com.yourapp`).
3. Download `google-services.json` → place in `mobile/` (Expo will copy it to `android/app/` at prebuild).
4. (Optional) Add an **iOS app** and download `GoogleService-Info.plist` if you need iOS push.
5. Generate a **Service Account** private key: Project Settings → Service Accounts → Generate new private key. This JSON is used by the server.

## 5. Server — initializing `firebase-admin`

Store the service account JSON as an environment variable (as a JSON string):

```
FIREBASE_SERVICE_ACCOUNT_JSON='{"type":"service_account","project_id":"...","private_key":"...","client_email":"...",...}'
```

Initialize once (lazy singleton):

```ts
import * as admin from "firebase-admin";

let _messaging: admin.messaging.Messaging | null = null;

function getMessaging(): admin.messaging.Messaging {
  if (_messaging) return _messaging;

  if (!admin.apps.length) {
    const serviceAccount = JSON.parse(process.env.FIREBASE_SERVICE_ACCOUNT_JSON!);
    admin.initializeApp({ credential: admin.credential.cert(serviceAccount) });
  }

  _messaging = admin.messaging();
  return _messaging;
}
```

## 6. Server — sending a push notification

**Key rule: use data-only messages** (no `notification` key). This forces Expo's native `ExpoFirebaseMessagingService` to handle and present the notification consistently, avoiding the system bypassing Expo on Android.

```ts
import { getMessaging } from "./firebase-admin"; // your singleton above

async function sendPush(fcmToken: string, payload: {
  title: string;
  body: string;
  data?: Record<string, string>;
}) {
  await getMessaging().send({
    token: fcmToken,
    data: {
      title: payload.title,
      message: payload.body,
      body: payload.body,
      channelId: "your_channel_id",
      sound: "default",
      ...payload.data,
    },
    android: { priority: "high" },
  });
}
```

All values in `data` must be strings.

### Handling stale tokens

FCM returns specific error codes for invalid/unregistered tokens. Clean them up:

```ts
const results = await Promise.allSettled(tokens.map((t) => sendPush(t, payload)));

const staleTokens: string[] = [];
results.forEach((result, i) => {
  if (result.status === "rejected") {
    const code = (result.reason as any)?.errorInfo?.code;
    if (
      code === "messaging/registration-token-not-registered" ||
      code === "messaging/invalid-registration-token"
    ) {
      staleTokens.push(tokens[i]);
    }
  }
});

// deactivate stale tokens in your DB
if (staleTokens.length) {
  await db.pushTokens.updateMany({
    where: { token: { in: staleTokens } },
    data: { active: false },
  });
}
```

## 7. Client — token registration hook

```ts
// hooks/usePushRegistration.ts
import * as Notifications from "expo-notifications";
import * as Device from "expo-device";
import { Platform } from "react-native";
import Constants from "expo-constants";

const IS_EXPO_GO = Constants.appOwnership === "expo";
const MAX_RETRIES = 5;

export async function registerForPushNotifications(): Promise<string | null> {
  // 1. Expo Go cannot receive FCM tokens
  if (IS_EXPO_GO) {
    console.warn("Expo Go — FCM not supported. Use a dev client or standalone build.");
    return null;
  }

  // 2. Physical device only
  if (!Device.isDevice) {
    console.warn("Push notifications require a physical device.");
    return null;
  }

  // 3. Request permission
  const { status: existingStatus } = await Notifications.getPermissionsAsync();
  let finalStatus = existingStatus;

  if (existingStatus !== "granted") {
    const { status } = await Notifications.requestPermissionsAsync();
    finalStatus = status;
  }

  if (finalStatus !== "granted") {
    console.warn("Push notification permission denied.");
    return null;
  }

  // 4. Create Android channel
  if (Platform.OS === "android") {
    await Notifications.setNotificationChannelAsync("your_channel_id", {
      name: "Your Channel Name",
      importance: Notifications.AndroidImportance.HIGH,
      sound: "default",
      vibrationPattern: [0, 200, 100, 200],
    });
  }

  // 5. Get FCM/APNs device token (with retry for transient FCM errors)
  for (let attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
      const tokenObj = await Notifications.getDevicePushTokenAsync();
      return tokenObj.data; // raw FCM token string
    } catch (err: any) {
      const isTransient =
        err?.message?.includes("SERVICE_NOT_AVAILABLE") ||
        err?.message?.includes("NETWORK_ERROR");

      if (isTransient && attempt < MAX_RETRIES) {
        await new Promise((res) => setTimeout(res, 1000 * attempt));
        continue;
      }
      throw err;
    }
  }

  return null;
}
```

> **Note:** Use `Notifications.getDevicePushTokenAsync()` (not `getExpoPushTokenAsync()`). The latter returns an Expo-proxied token that requires routing through Expo's push service; the former gives the raw FCM/APNs token that `firebase-admin` can use directly.

## 8. Client — sending the token to your backend

After obtaining the token, register it with your server so it can target the correct device:

```ts
await fetch("/api/push/register", {
  method: "POST",
  headers: { "Content-Type": "application/json", Authorization: `Bearer ${jwt}` },
  body: JSON.stringify({
    token: fcmToken,
    platform: Platform.OS, // "android" | "ios"
  }),
});
```

On logout, unregister the token (best-effort, fire-and-forget):

```ts
await fetch("/api/push/unregister", {
  method: "POST",
  headers: { "Content-Type": "application/json", Authorization: `Bearer ${jwt}` },
  body: JSON.stringify({ token: cachedToken }),
}).catch(() => {});
```

## 9. Client — foreground and tap handling (`App.tsx`)

### Foreground display

Call this once at module level (outside any component):

```ts
import * as Notifications from "expo-notifications";

Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: false,
    shouldShowBanner: true,
    shouldShowList: true,
  }),
});
```

### Tap / action response

Wire this inside your root component:

```ts
useEffect(() => {
  const sub = Notifications.addNotificationResponseReceivedListener((response) => {
    const data = response.notification.request.content.data as Record<string, string> | undefined;
    const actionId = response.actionIdentifier;

    // Handle custom action buttons
    if (actionId === "some_action" && data?.someId) {
      // call your API...
      return;
    }

    // Default tap → navigate somewhere
    if (data?.screen === "SomeScreen") {
      navigationRef.current?.navigate("SomeScreen" as never);
    }
  });

  return () => sub.remove();
}, []);
```

## 10. Database schema (Prisma)

```prisma
model PushToken {
  id         String   @id @default(cuid())
  userId     String   @map("user_id")
  token      String
  platform   String   @default("android")  // "android" | "ios"
  active     Boolean  @default(true)
  lastSeenAt DateTime @default(now()) @map("last_seen_at")
  createdAt  DateTime @default(now()) @map("created_at")
  updatedAt  DateTime @updatedAt @map("updated_at")

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, token])
  @@index([active])
  @@map("push_tokens")
}

model User {
  // ...
  pushNotificationsEnabled Boolean @default(false) @map("push_notifications_enabled")
}
```

## 11. API routes (Next.js example)

### `POST /api/push/register`

```ts
import { prisma } from "@/lib/prisma";

export async function POST(req: Request) {
  const auth = await getAuthFromRequest(req); // your auth helper
  const { token, platform } = await req.json();

  await prisma.pushToken.upsert({
    where: { userId_token: { userId: auth.id, token } },
    update: { platform, active: true, lastSeenAt: new Date() },
    create: { userId: auth.id, token, platform },
  });

  return Response.json({ ok: true });
}
```

### `POST /api/push/unregister`

```ts
export async function POST(req: Request) {
  const auth = await getAuthFromRequest(req);
  const { token } = await req.json();

  await prisma.pushToken.updateMany({
    where: { userId: auth.id, token },
    data: { active: false },
  });

  return Response.json({ ok: true });
}
```

### `POST /api/push/preferences`

```ts
export async function POST(req: Request) {
  const auth = await getAuthFromRequest(req);
  const { pushNotificationsEnabled } = await req.json();

  await prisma.user.update({
    where: { id: auth.id },
    data: { pushNotificationsEnabled },
  });

  // Deactivate all tokens when disabling
  if (!pushNotificationsEnabled) {
    await prisma.pushToken.updateMany({
      where: { userId: auth.id },
      data: { active: false },
    });
  }

  return Response.json({ ok: true });
}
```

## 12. Checklist for a new project

- [ ] Create Firebase project + Android/iOS apps
- [ ] Download `google-services.json` (and `GoogleService-Info.plist` for iOS) → place in `mobile/`
- [ ] Add `expo-notifications` (and optionally `expo-device`) to `mobile/package.json`
- [ ] Configure `expo-notifications` plugin in `app.json` (icon, color, defaultChannel)
- [ ] Set `googleServicesFile` path in `app.json` under `android` (and `ios` if applicable)
- [ ] Add `firebase-admin` to server `package.json`
- [ ] Store service account JSON in `FIREBASE_SERVICE_ACCOUNT_JSON` env var
- [ ] Initialize `firebase-admin` singleton on the server
- [ ] Create DB table for push tokens (see schema above)
- [ ] Implement `/api/push/register` and `/api/push/unregister` routes
- [ ] Call `registerForPushNotifications()` on app load (after auth, if push is enabled)
- [ ] POST the token to `/api/push/register`
- [ ] Set `Notifications.setNotificationHandler(...)` at module level in `App.tsx`
- [ ] Add `addNotificationResponseReceivedListener` in root component for tap handling
- [ ] On logout, call `/api/push/unregister` before clearing auth state
- [ ] Build with `expo-dev-client` or `eas build` — **never test push in Expo Go**
- [ ] Send a test push from server to verify end-to-end

## 13. Notes & gotchas

- **Data-only vs notification messages:** Always use the `data` key only on FCM messages. If you include a `notification` key, Android will bypass Expo's native handler and the JS layer won't receive it reliably.
- **Raw FCM token vs Expo push token:** Use `getDevicePushTokenAsync()` for the raw token (works directly with `firebase-admin`). `getExpoPushTokenAsync()` routes through Expo's proxy service and is not needed when using `firebase-admin` directly.
- **iOS background delivery:** Add `UIBackgroundModes: ["remote-notification"]` in `app.json` under `ios.infoPlist` if you need background delivery and silent pushes.
- **Token rotation:** FCM tokens can change. Re-register on every app launch (the upsert approach handles this gracefully).
- **Stale token cleanup:** FCM returns `messaging/registration-token-not-registered` for dead tokens — deactivate them in your DB immediately.
- **Android 13+:** `POST_NOTIFICATIONS` is a runtime permission. `expo-notifications` will request it automatically.
- **Testing on simulator:** iOS simulator does not support push notifications. Android emulator can receive FCM if Google Play Services is present, but real devices are always more reliable.
